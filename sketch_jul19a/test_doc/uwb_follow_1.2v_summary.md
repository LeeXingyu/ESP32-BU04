# `uwb_follow_1.2v.py` 方案总结

## 1. 方案目标

`uwb_follow_1.2v.py` 用于实现基于 UWB 标签的机器人跟随。

这版方案的核心目标不是连续速度控制，而是：

- 持续从 BU04/UWB 串口读取相对坐标
- 将标签相对坐标转换为地图坐标
- 以 `FollowPathPointsAction` 的方式批量下发路径点
- 通过滤波、批处理和实时更新减少“走两步停一下”的现象
- 在距离过近或角度异常时暂停导航，避免机器人乱转或冲向目标


## 2. 总体思路

这版方案保持“直接追标签”的策略，不生成安全跟随点。

整体流程如下：

1. 串口持续读取 UWB 数据
2. 对 `Xcm/Ycm` 做 EMA 平滑和跳变钳制
3. 结合机器人当前位姿，将标签坐标转换到地图坐标
4. 将采样点按序号放入队列
5. 使用 `FollowPathPointsAction` 批量发送多个目标点
6. 在满足条件时执行实时更新：取消当前路径并拼接新路径重发
7. 当距离过近或角度异常时暂停导航


## 3. 数据流

### 3.1 输入

- 串口 UWB 数据
  - 来源：BU04
  - 格式：代码当前按 `{"TWR": {"Xcm": ..., "Ycm": ...}}` 解析
- 机器人位姿
  - 来源：机器人 API
  - 内容：`x, y, yaw`

### 3.2 输出

- 调试输出
  - 原始 UWB 值
  - 滤波后 UWB 值
  - 角度信息
  - 地图坐标
- 导航输出
  - `FollowPathPointsAction`
  - 取消当前路径
  - 查询当前 action 是否运行中


## 4. 核心模块

### 4.1 坐标转换

通过 `tag_to_map()` 将标签相对小车的坐标转换为地图坐标：

- `Xcm`：标签在小车右侧的距离
- `Ycm`：标签在小车前方的距离
- `yaw`：小车在地图坐标系中的朝向

这意味着当前方案是“直接追标签位置”，而不是“追一个安全偏移点”。


### 4.2 UWB 滤波

当前使用两级抑制手段：

- 跳变钳制
  - 如果当前原始点相对上一帧滤波值的跳变超过阈值，则将该跳变截断到阈值以内
- EMA 平滑
  - 使用 `filtered = alpha * raw + (1 - alpha) * prev_filtered`

作用：

- 抑制 BU04 单帧飞点
- 降低高频抖动
- 让路径点更平滑

当前默认参数：

- `EMA_ALPHA = 0.3`
- `CLAMP_MAX_JUMP_CM = 50.0`


### 4.3 队列与批量路径

采样点会按 `(seq, x, y)` 入队：

- `seq` 单调递增
- 队列只保留最新 `MAX_QUEUE_SIZE` 个点
- 发送时只取“上次发送后新增”的点

这样做的目的：

- 不重复发送旧点
- 让路径始终跟随最新采样结果
- 避免队列无限变大

默认参数：

- `MAX_QUEUE_SIZE = 30`
- `MIN_FIRST_BATCH = 10`
- `MAX_BATCH_SIZE = 30`


### 4.4 实时更新模式

开启 `--live` 后，方案会在当前 `FollowPath` 运行中执行“取消 + 拼接 + 重发”。

策略：

1. 获取机器人当前位置
2. 从当前批次中找出最近点
3. 保留该点之后的剩余路径作为“过渡段”
4. 追加新的采样点
5. 取消旧路径
6. 下发新路径

目标：

- 尽量减少机器人等待当前批次完全结束
- 让路径随着标签移动持续更新

默认触发条件：

- `LIVE_UPDATE_NEW_POINTS = 5`

也就是积累到 5 个新点后触发一次实时更新。


### 4.5 距离安全策略

当前采用四档速度区间：

- `< 0.5m`：暂停导航
- `0.5m ~ 1.0m`：减速
- `1.0m ~ 2.5m`：正常
- `> 2.5m`：加速

意义：

- 目标太近时不再继续逼近
- 距离中等时放慢
- 距离过大时提高追赶速度


### 4.6 角度异常保护

新增“角度异常时只暂停不改目标”的保护。

判定方式：

- 使用 `atan2(xcm, ycm)` 计算标签相对车头角度
- 当 `|angle| > MAX_TRACK_ANGLE_DEG` 时视为异常
- 当 `ycm <= 0` 时，视为标签已到侧后方，也判为异常

保护动作：

- 取消当前 `FollowPath`
- 暂停发送新批次
- 不清空队列
- 不重置序号
- 不改变“直接追标签”的总体策略

恢复方式：

- 角度回到恢复阈值以内后，恢复导航
- 当前使用滞回：
  - 暂停阈值：`65°`
  - 恢复阈值：`55°`


## 5. 运行模式

### 5.1 普通模式

特点：

- 正常采样
- 正常发批量路径
- 支持距离安全控制
- 可选实时更新


### 5.2 `--live` 实时更新模式

特点：

- 当前路径运行中也会周期性取消并重发
- 可以减少“等整批走完再更新”的迟滞
- 但也更依赖底盘对取消/重发的稳定支持


### 5.3 `--dry-run` 调试模式

特点：

- 只打印 UWB、滤波、角度和地图坐标
- 不发送任何导航指令
- 适合验证：
  - 串口数据是否正确
  - 坐标变换是否合理
  - 角度保护是否按预期触发


## 6. 默认参数一览

### 6.1 采样与队列

- `SAMPLE_INTERVAL = 0.2`
- `MAX_QUEUE_SIZE = 30`
- `MIN_FIRST_BATCH = 10`
- `MAX_BATCH_SIZE = 30`

### 6.2 滤波

- `EMA_ALPHA = 0.3`
- `CLAMP_MAX_JUMP_CM = 50.0`

### 6.3 距离安全

- `TOO_CLOSE_DISTANCE = 0.5`
- `SLOW_DISTANCE = 1.0`
- `FAST_DISTANCE = 2.5`
- `SLOW_SPEED_RATIO = 0.3`
- `NORMAL_SPEED_RATIO = 1.0`
- `FAST_SPEED_RATIO = 1.2`

### 6.4 角度保护

- `MAX_TRACK_ANGLE_DEG = 65.0`
- `ANGLE_RECOVER_GAP = 10.0`


## 7. 方案优点

- 保持“直接追标签”的业务逻辑，符合当前控制目标
- 相比单点 `MoveToAction`，`FollowPathPointsAction` 更连续
- EMA + 跳变钳制可明显减弱 UWB 抖动
- 队列和序号机制可以避免重复发送旧点
- `--live` 模式可以进一步降低停顿感
- 角度异常保护可避免标签跑到大侧角或侧后方时机器人继续乱追


## 8. 当前已知局限

### 8.1 仍然依赖点到点路径接口

虽然用了批量路径，但底层仍不是连续速度控制。

这意味着：

- 无法做到真正“丝滑跟人”
- 仍可能存在某种程度的停顿或重规划顿挫


### 8.2 直接追标签会放大坏数据风险

因为目标就是标签位置本身：

- 如果 BU04 出现系统偏差，目标点会直接偏
- 如果标签靠墙、出门框或贴边，地图点也可能危险

这版通过滤波和角度保护在“减风险”，但没有从策略上回避这个问题。


### 8.3 当前没有地图可达性校验

也就是说：

- 代码当前不会判断目标点是否落在墙体外
- 不会检查该点是否真的可走

如果后续需要更稳，最好补一层：

- 地图边界限制
- 墙体/障碍区过滤
- 不可达点直接丢弃


### 8.4 距离安全阈值仍是硬切换

当前距离区间是单阈值切换。

这意味着在临界区附近可能发生：

- `too_close` / `slow` 来回切换
- `slow` / `normal` 来回切换
- `normal` / `fast` 来回切换

如果实机上发现速度档位频繁抖动，建议后续补“滞回区间”。


### 8.5 串口异常流保护仍可继续加强

当前串口读取是基于 JSON 片段解析。

如果出现长期异常流、坏帧或未闭合 JSON，建议继续补：

- 缓冲上限
- 无效前缀丢弃
- 超长缓存主动清理


## 9. 推荐使用方式

### 9.1 先做离线验证

先用：

```bash
python uwb_follow_1.2v.py COM10 --dry-run
```

观察：

- 原始 UWB 数据是否稳定
- 滤波后是否明显更平滑
- 角度值是否合理
- 地图坐标是否随标签移动变化正确


### 9.2 再做普通跟随测试

```bash
python uwb_follow_1.2v.py COM10
```

先看：

- 是否能稳定形成批量路径
- 是否仍然“走两步停一下”
- 距离过近时是否能及时暂停


### 9.3 最后再尝试实时更新模式

```bash
python uwb_follow_1.2v.py COM10 --live --live-interval 5
```

适合在下面情况试用：

- 标签移动较连续
- 机器人取消+重发 FollowPath 的能力较稳定
- 希望进一步降低路径更新滞后


## 10. 后续可优化方向

如果继续迭代，建议优先级如下：

1. 地图可达性/墙体外点过滤
2. 距离档位滞回
3. 串口缓冲异常保护
4. 对角度保护增加持续时间统计和恢复稳定判定
5. 根据现场调参优化：
   - `EMA_ALPHA`
   - `CLAMP_MAX_JUMP_CM`
   - `MIN_FIRST_BATCH`
   - `LIVE_UPDATE_NEW_POINTS`
   - `MAX_TRACK_ANGLE_DEG`


## 11. 一句话总结

`uwb_follow_1.2v.py` 的本质是：

**在“直接追标签”的前提下，用 EMA 滤波、批量 FollowPath、实时更新和角度保护，尽量把 UWB 跟随做得更连续、更稳、更不容易乱转。**
